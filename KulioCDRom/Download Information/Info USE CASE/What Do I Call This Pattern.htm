Here's the fix to the r:\ path problem, sent in by Rob Fowler.

Copy both UseCases.mdb and UsecasesBE.mdb to the same directory after unzipping.

1. While opening the UseCase.mdb from Access, hold down the shift key. This keeps the application from running. ( I opened it in Access 97 and converted it. The fix might work in Access 2.0 differently.)

2. Open the Linked Table Manager menu item from menu path: Tools.Add-Ins

3. Check the Radio Button at bottom of dialog box: Always Prompt for New Location. Click on Select All , OK

4. Once dialog box is open, highlight the UseCaseBE.mdb file. Then Click on Open.

5. Once all your files are re-linked to the correct directory, you can close the dialog box.

6. Close and Reopen to run application or open the form mnuMainMenu under forms tab.

I ran a couple of screens and they came up fine. Let me know how well it works as a use case tool.
