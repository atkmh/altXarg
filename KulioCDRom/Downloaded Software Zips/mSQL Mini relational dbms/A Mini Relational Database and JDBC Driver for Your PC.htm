A Mini Relational Database and JDBC Driver for Your PC

by Don Denoncourt

In 1997, when I first set out to learn how to access relational database files using Java Database Connect (JDBC), the first text I read was Database Programming with JDBC and Java by George Reese. I had always been a proponent of SQL and as a Java evangelist JDBC seemed an elegant strategy to access DB2/400. My development environment, however, was a laptop and I often coded in places where an Internet connection for AS/400 access was not available. (Places such as a park bench or on a lawn chair in my backyard, or, curse the thought, at my mother-in-law’s house during a family visit.) Reese’s book proposed a solution to my AS/400 inaccessibility problem--a lightweight workstation-based relational database environment. Reese, you see, is the developer of the world’s first JDBC driver (mSQL-JDBC). His JDBC driver was for the Hughes Technologies Pty Ltd.'s (www.hughes.com.au/) Mini SQL (mSQL) relational database. But back in 1997, the mSQL database engine did not work on Windows. A few weeks ago, I discovered that mSQL now works on Windows 9x/NT. Further, I discovered that not only was there a JDBC driver for mSQL, but there was also a 100% pure Java GUI for mSQL.

In this article, I will show you how to download, configure, and test mSQL, the Java-based GUI, and the JDBC driver. But before I do that, I should mention that mSQL is available on a "14-day evaluation period." At the end of that period, David J. Hughes, managing director of Hughes Technologies expects you to send in the registration form provided with the documentation. Note, however, that some groups may continue to use mSQL free of charge as Hughes points out in mSQL’s readme file:

Hughes Technologies continues to support the original ethos of the Internet by providing mSQL 2.0 (and all other Hughes Technologies products) free of charge to certain organisations. If your use of our software is in conjunction with your work or study at one of the organisational types listed below you may continue to use the software beyond the 14 day evaluation without licensing your copy.

  • Universities and schools
  • Non-commercial research organisations
  • Registered not-for-profit, organisations
  • Registered charities
  • Churches

Download and Setup

To download mSQL, the Java-based GUI, and the JDBC driver visit blnet.com/msqlpc/ (the Hughes’ site only has the mSQL engine; this site has all the products) and select the following products for download:

  • mSQL 2.0.4.1 Build #9 for Win32 - CygWin build
  • GUImSQL client
  • JDBC Driver 2.0b4 for mSQL 1.0.x and 2.0.x (JDK 1.2)
  • JDBC Driver 1.0 for mSQL 1.0.x and 2.0.x (JDK 1.1.x)

The version numbers may have changed, so I’d suggest you download the most recent versions that are not in beta. Also, as Java proponents, I’d expect you to do as I did and download the "Pure Java" build of GUImSQL not the "Native Build." The pure Java build requires that you have a JDK installed on your machine to be able to execute the Java classes. Note too that there are JDBC drivers for both Java 1.1.x and Java 1.2, so download the JDBC driver for whichever JDK version you are using. For various reasons, I use both JDK 1.1 and JDK 1.2, so I downloaded both drivers.

For best results, create a directory called C:\usr and C:\usr\local. (It’s a UNIX thing.) Then unzip the msql-2041-build9.zip to C:\usr\local. The C drive is the default so if you don’t/can’t use C: you’ll have to modify the msql.conf configuration file’s Inst_Dir property. Next unzip the Java GUI (GuiMSQL.zip) to some directory. On my machine I simply put it under C:\usr\local. Finally unzip the JDBC driver (mSQL-JDBC-2b.zip) to wherever. Just remember that wherever you put it you’ll have to add the location of the JAR (msql-jdbc-2-0b5.jar) file to your CLASSPATH.

Start Your Engines

To start the mSQL server, open a DOS window, change the current directory to c:\usr\local\hughes\bin (or just have c:\usr\local\hughes\bin as a part of your PATH environment variable) and execute the msql2d.exe Windows program. The d is for daemon, which is what UNIX geeks commonly call server programs because they quietly lie waiting and then pounce up in a flurry of activity in response to a user entry. Typically HTTP server programs (such as the Apache HTTP server) are called httpd.exe. At this point the mSQL driver is lying in wait for SQL access on the default port of 1114.

The mSQL system comes with a database called joshua, but you’ll want to set up your own (or several for that matter). To create your own application database, use mSQL’s administration program, msqladmin.exe. For instance, I used the following command to create a database called denoncourt:

C:\usr\local\Hughes\bin>msqladmin create denoncourt

Note that the mSQL administration program created a directory called denoncourt within the msqldb directory of the Hughes directory (c:\usr\local\msqldb\denoncourt).

To add a table to your newly created database, you could use a Java program and the msql JDBC driver, but creating tables is best done (or, at least, more easily done) with a GUI. So now it’s time to fire up your Java mSQL GUI. Open a new DOS window and change your current directory to wherever you decompressed GuiMSQL.zip. Then fire up the GUImsql AWT Java application:

C:\usr\msqlgui>java GUImsql

You should see a Java GUI like the one shown in Figure 1. Change the DB UserName to whatever you want; I simply keyed in my last name. The mSQL database enables security through access control lists (ACL), but the default setting is ACL-disabled (which, in fact, the mSQL engine warns you about on startup). As you can see in Figure 2, I selected localhost for the DB HostName and denoncourt for the DB UserName. If all goes well (meaning your msql2d.exe database engine is appropriately configured and currently running in a separate window), you should see a successful connect message at the bottom of the mSQL GUI.

With the mSQL GUI successfully connected, your next step is to select your database. Click the Pick DB button and from the Choose the Database panel (as shown on Figure 3), select the database you wish to work with, and click OK. I selected my denoncourt database.

Now you are ready to create a file. The SQL command to create a file is CREATE TABLE and, as you can see in Figure 4, I created a file called salesperson with the following SQL command:

CREATE TABLE salesperson (salesId INT, salesman CHAR (20))

Once you complete typing your CREATE TABLE command you ask mSQL to execute your SQL statement with the Run Query button. Hopefully you’ll get a message--Query executed successfully!--in the bottom of the display.

Next you’ll want to populate your database. Again, you could do this with a Java program but it is handy to have a GUI environment if only to test the syntax of your SQL statements. I added two records to my file with SQL’s INSERT statement:

INSERT INTO salesperson VALUES (1, 'Don Denoncourt')

INSERT INTO salesperson VALUES (2, 'Joe Pluta')

To list the records of my salesperson file, I cleared the mSQL GUI with the Clear button and then typed the following SQL SELECT statement:

SELECT * FROM salesperson

The mSQL GUI then presented the results of my SELECT statement (two records) in the middle panel of its GUI.

mSQL Access with JDBC

Finally, with my database file populated with records, I was able to test the mSQL JDBC driver. Figure 5 shows my Java application that simply lists the contents of my salesperson file. The code for my ListMSQLFile class is self-explanatory, so I only have to point out the URL qualification for the mSQL engine. ListMSQLFile, as you would expect, loads the mSQL JDBC driver (which George Reese called com.imaginary.sql.msql.MsqlDriver. (Can you imagine having a real company called Imaginary?) Then, when a JDBC Connection object is retrieved with the DriverManager class’s getConnection method, the protocol, as always, is JDBC, but the subprotocol is msql. The URL then specifies the IP address, which for local access is 127.0.0.1 followed by the port number, which, if you remember, defaults to 1114. It is very important that you also note the URL qualifies the name of the database following the IP address and the port number:

DriverManager.getConnection("jdbc:msql://127.0.0.1:1114/denoncourt");

After a Connection object is successfully returned, the ListMSQLFile application simply dumped the records (all two of them) to the AWL list box shown in Figure 6.

mSQL Potential
My example Java application may not be too impressive, but the three products that it uses are indeed impressive. The mSQL database has been proven to be a stable enough for commercial development. It does, however, lack some of the more sophisticated features (such as commitment control) that are available with other, more expensive, relational databases like DB2 UDB. Nevertheless, many small companies are using mSQL as a low-cost database server for their Internet applications.
